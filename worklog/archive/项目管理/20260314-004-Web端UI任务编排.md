# Web 端 UI 任务编排

- 记录时间：2026-03-14
- 用户原始诉求：根据讨论内容，进行 UI 任务编排
- 当前状态：讨论中（未进入代码修改）。

## 本轮讨论目标
- 基于既有 Web 端实现方案与 UI 设计结论，拆出一份可执行的 UI 实施任务编排。
- 明确任务依赖、阶段边界、收口门禁与延后项，避免实现时顺序混乱。
- 形成后续进入代码修改阶段时可直接采用的任务清单。

## 当前已知约束
- 本轮属于项目问题，需先记录并进入讨论。
- 未获得用户明确同意前，不执行代码修改。
- Web 第一阶段 UI 已收敛为“单页任务控制台”，不是类聊天产品。
- 第一阶段前端形态已冻结为 `单页薄界面 + 原生表单/轻量 JS`。
- 会话真相源必须仍为 `STM`，前端只维护瞬时展示状态。
- 首版必备能力已冻结为：提交、续写、`clarify`、状态查看、重置。

## Team 分工（架构/实现/测试/风险）
- 架构：按依赖关系编排 UI 阶段任务，避免越层实现。
- 实现：拆分页面骨架、控件、状态反馈与会话展示任务。
- 测试：为每个阶段绑定最小验证点，避免末端一次性堆积风险。
- 风险：识别容易膨胀或与 API/STM 边界冲突的任务，并明确延后。

## 共识结论（是否通过 + 条件）
- 结论：通过，可以将首版 UI 任务编排为“页面骨架 -> 表单闭环 -> 会话展示 -> 状态反馈 -> 响应式收口 -> 门禁测试 -> 文档改口”七段顺序。
- 通过条件：
  - UI 任务必须以后端 API 契约和共享服务层为前置依赖，不允许前端先发散设计。
  - 首版任务只覆盖既定六大页面区块：顶栏、任务配置、状态卡片、结果摘要、历史面板、会话控制。
  - 每一阶段都要绑定最小验收点，未通过前不进入下一阶段。
  - 增强项必须单列延期，不得混入首版任务。

## 风险清单（高/中/低）
- 高：
  - 若在页面骨架阶段就引入多会话、富文本、流式输出，会破坏任务边界。
  - 若在表单阶段由前端自行推断业务字段规则，会侵入服务层职责。
  - 若在会话展示阶段把前端历史当真相源，会与 `STM` 边界冲突。
- 中：
  - 若不先绑定阶段验收点，容易在末尾集中暴露大量 UI 状态问题。
  - 若桌面与移动布局并行发散设计，会拖慢首版收敛速度。
  - 若测试放到最后一次性补齐，回归定位成本会明显上升。
- 低：
  - 帮助提示、样式微调、视觉强化延后，风险可控。

## 执行顺序（Phase 1..N）
- Phase 1：前置依赖冻结
  - 确认共享入口服务层与 Web API 契约已冻结。
  - 冻结页面六大区块及字段清单，不再讨论功能增项。
  - 验收点：字段、区块、API 映射表可直接指导 UI 实现。

- Phase 2：页面骨架与布局
  - 搭建顶栏、左右两栏主框架；移动端退化为单栏顺序布局。
  - 固定六大区块占位：顶栏、任务配置、状态卡片、结果摘要、历史面板、会话控制。
  - 验收点：桌面与移动端首屏均可到达关键操作区，无关键控件丢失。

- Phase 3：任务配置表单闭环
  - 实现 `goal`、`action_type`、条件化 `target`、发送、清空输入。
  - 补齐字段显隐、必填约束、提交禁用、防重复点击。
  - 验收点：用户可完成一次最小提交流，参数发出符合 API 契约。

- Phase 4：结果区与历史区闭环
  - 实现结果摘要区，展示最近一轮输出、响应类型、关键摘要。
  - 实现历史面板，按顺序渲染用户/系统/Agent 消息。
  - 绑定同会话连续两轮追加逻辑，不覆盖旧记录。
  - 验收点：`基础提交流` 与 `会话续写流` 可跑通。

- Phase 5：状态反馈与会话控制
  - 实现加载态、空态、字段错误、后端错误、`clarify` 引导态。
  - 实现 `session_id` 展示、复制、刷新状态、重置会话。
  - 为重置增加确认机制，确保映射真实后端重置能力。
  - 验收点：`clarify/错误映射流` 与 `重置流` 可跑通。

- Phase 6：响应式与可用性收口
  - 收敛桌面端“两栏 + 顶栏”和移动端“单栏”布局。
  - 修正滚动、按钮尺寸、输入间距、状态区可见性。
  - 验收点：窄桌面视口与移动端 smoke 下首屏可操作，关键状态可见。

- Phase 7：测试门禁与文档改口
  - 补齐 UI 最小门禁：
    - `基础提交流`
    - `会话续写流`
    - `clarify/错误映射流`
    - `重置流`
  - 同步调整 README/说明文案中的入口描述。
  - 验收点：UI 门禁可替代 CLI 入口门禁参与正式收口。

## 延后项（不进入首版）
- 多会话列表、多标签页、会话搜索。
- WebSocket、流式输出、逐 token 渲染。
- 富文本消息、Markdown 增强、代码高亮、附件上传。
- 复杂调试面板、执行轨迹树、插件/知识侧栏。
- 深色模式完整适配、主题切换、个性化皮肤。
- 完整前后端分离 SPA 与重前端状态管理。
