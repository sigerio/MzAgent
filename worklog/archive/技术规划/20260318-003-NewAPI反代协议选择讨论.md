# NewAPI 反代协议选择讨论

- 记录时间：2026-03-18
- 用户原始诉求：如果均走newapi反代，内部在使用不同模型的时候，是使用openai的协议还是 Anthropic的协议
- 当前状态：进入方案讨论，未执行代码修改。

## 背景现状

- 当前项目内部 LLM 调用链统一基于 OpenAI Python SDK。
- 当前实际请求路径是 `client.responses.create(...)`，属于 OpenAI Responses API 风格。
- 用户希望确认：如果上游统一走 NewAPI 反代，那么当模型来自不同供应商时，内部是否仍可统一走 OpenAI 协议。

## 本轮讨论目标

1. 确认 NewAPI 对外暴露的是单一协议还是多协议。
2. 确认在 NewAPI 下接入 Anthropic 模型时，是否必须使用 Anthropic 原生协议。
3. 评估当前项目继续统一使用 OpenAI 协议的边界与影响。

## 用户新增约束

- 不再接入官方 API Key。
- 当前项目后续统一走 NewAPI 提供的 `base_url + api_key + model name`。
- 当前讨论的重点从“是否支持多供应商直连”收敛为：
  - 在统一走 NewAPI 的前提下，内部协议是否继续统一为 OpenAI 风格。
