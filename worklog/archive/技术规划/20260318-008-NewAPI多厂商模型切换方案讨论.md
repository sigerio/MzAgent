# NewAPI 多厂商模型切换方案讨论

- 记录时间：2026-03-18
- 用户原始诉求：同一 NewAPI 站点下可能同时提供 GPT 与 Claude 等不同厂商模型，用户希望在对话后手动切换模型，或让 MzAgent 自动切换到另一模型。
- 当前状态：已完成方案讨论与实现，当前版本按模型保存协议字段，并在运行时按协议分发调用链。

## 背景现状

- 当前项目已切换为单 NewAPI 连接 + 多模型条目 + 单模型启用。
- 当前运行时配置中 `api_mode` 固定为 `responses`。
- 当前 LLM 适配器真实调用路径固定为 `client.responses.create(...)`。
- 因此当前系统虽然可以保存多个模型名，但并未具备“按模型协议差异动态切换请求格式”的能力。

## 本轮讨论焦点

1. 手动切换模型时，是否只切换“当前启用模型”，还是允许单轮对话临时指定模型。
2. 自动切换模型时，切换依据是用户指令、失败回退，还是任务类型路由。
3. 同一 NewAPI 站点下不同模型若协议要求不同，是否需要为每个模型保存额外的调用元数据。

## 实施结果

- 已参照 `openclaw` 的 provider/model 思路，将 MzAgent 的模型配置从“仅模型名”扩展为“模型名 + 协议模式”。
- 已支持三种协议模式：
  - `openai-responses`
  - `openai-completions`
  - `anthropic-messages`
- 已在设置页添加模型协议选择项，并在已添加模型列表与连接测试结果中展示当前协议。
- 已将运行时调用链改为按模型协议分发：
  - `openai-responses` -> `/v1/responses`
  - `openai-completions` -> `/v1/chat/completions`
  - `anthropic-messages` -> `/v1/messages`
