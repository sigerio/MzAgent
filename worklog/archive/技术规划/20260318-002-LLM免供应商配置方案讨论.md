# LLM 免供应商配置方案讨论

- 记录时间：2026-03-18
- 用户原始诉求：再讨论一下，LLM连接方案，如何实现不需要用户指定供应商，仅需要用户提供base url，api key 和model name即可
- 当前状态：进入方案讨论，未执行代码修改。

## 背景现状

- 当前 `LLMProfile` 仍要求显式保存 `provider_type`。
- 当前 `provider_type` 仅有两类：`openai_native` 与 `openai_compatible_proxy`。
- 当前配置有效性判断依赖 `provider_type`：
  - `openai_native`：只要求 `model` 与 `api_key`
  - `openai_compatible_proxy`：额外要求 `base_url`
- 用户现在希望前端与配置体验中不再要求“指定供应商”，而是只填：
  - `base_url`
  - `api_key`
  - `model name`

## 本轮讨论目标

1. 讨论是否保留内部 `provider_type` 概念。
2. 讨论如何让用户侧输入变成“无供应商感知”。
3. 讨论仅靠 `base_url`、`api_key`、`model name` 是否足够驱动当前 LLM 调用。
4. 讨论兼容历史 profile 数据时的过渡策略。

## 用户新增约束

- 不再接入官方 OpenAI API Key。
- 后续统一使用 NewAPI 提供的：
  - `base_url`
  - `api_key`
  - `model name`
- 当前项目的 LLM 接入目标收敛为：
  - 仅支持通过 NewAPI 网关访问模型
  - 不再提供“官方直连”和“供应商选择”两套路径
