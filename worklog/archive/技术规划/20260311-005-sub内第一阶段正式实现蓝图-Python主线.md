## 风险清单（高/中/低）
- 高：若不及时把旧 Rust 蓝图从 active 主线挪开，后续讨论会出现双主线并存。
- 高：若 Python 主线不先重建契约测试门禁，协议漂移和状态漂移会迅速放大。
- 高：若直接复用参考框架对象而不先重建主协议层，主协议会被供体反向定义。
- 中：Python 让首刀样板更少，但也更容易放松动态字段、空值表达和异常语义的刚性。
- 中：若一开始就讨论 `tool / mcp / rag / llm / skill` 具体接入，会冲掉“先门禁、后功能”的节奏。
- 低：后续若要再引入更硬内核或混合路线，仍可讨论，但当前一阶段不应展开。
