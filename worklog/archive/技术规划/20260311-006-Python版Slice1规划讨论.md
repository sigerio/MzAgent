## Pydantic 承载基线

### 统一模型基线
- `contracts/*` 下的对象模型统一：
  - 继承 `BaseModel`
  - 显式声明字段
  - 默认使用 `model_validate`
  - 对外导出时统一使用 `model_dump`
- 推荐基础配置：
  - `model_config = ConfigDict(extra='forbid')`
- 目的：
  - 防止“顺手多塞字段”导致协议漂移

### 动态字段承载策略
- 当前这些字段必须允许结构化但受控的动态值：
  - `perception`
  - `stm`
  - `skill_context`
  - `last_observation`
  - `action_input`
- 最终首刀策略：
  - `perception / stm / skill_context / action_input` 统一为 `dict[str, object]`，默认 `{}`，使用 `default_factory=dict`
  - `last_observation` 统一为 `dict[str, object] | None`，默认 `None`
  - 5 个字段全部禁止裸 `Any`
  - 不在 `Slice 1` 里过早细化成大量嵌套子模型
- 原因：
  - 要兼顾首刀速度
  - 但不能完全失去边界

### 状态与决策承载
- `StepState`
- `CursorState`
- `ReactStatus`
- `GuardrailsDecisionKind`
- 统一使用：
  - `Enum`
- 不使用：
  - 普通字符串常量散落在各模块

## runtime 层基线

### `runtime/errors.py`
- 首批只收敛：
  - `ContractError`
  - `TransitionError`
  - `ValidationError`（领域层，不与 Pydantic 同名导出混淆时需谨慎命名）
- 更稳的命名建议：
  - `ContractViolationError`
  - `TransitionViolationError`
  - `DomainValidationError`
- 目标：
  - 与 Pydantic 的错误面区分开

### `runtime/transitions.py`
- 必须保留独立纯函数，不把状态门禁塞进模型 validator：
  - `validate_step_transition`
  - `validate_cursor_transition`
  - `validate_react_transition`
  - `invalidate_current_step_on_plan_change`
- 最小契约：
  - 前 3 个函数成功时返回目标状态，非法转换抛 `TransitionViolationError`
  - `invalidate_current_step_on_plan_change` 在版本未变时返回原 `current_step`，版本变化时返回 `None`
- 固定边界：
  - 只处理状态门禁与版本失效
  - 不处理对象修补、结果映射、编排调度或 `STM` 回写
- 原因：
  - 这样 `state_transitions` 测试才能保持门禁测试而不是流程测试

## 风险清单（高/中/低）
- 高：如果 `contracts/*` 不统一启用 `extra='forbid'`，Python 下协议漂移会比 Rust 更快暴露。
- 高：如果把状态门禁写进 Pydantic validator 而不是独立纯函数，测试会很快退化成对象校验混合测试。
- 高：如果动态字段直接放成裸 `Any`，首刀虽然快，但会马上失去协议边界。
- 中：如果 `runtime/errors.py` 直接复用 Pydantic 的 `ValidationError` 语义，领域错误和模型错误会混在一起。
- 中：如果首刀就引入 `pytest-asyncio`、provider SDK 或 tracing 依赖，讨论会立刻滑向 Slice 2/3。
- 低：后续可能需要把动态字段再细化成子模型，但当前先保持受控动态承载更稳。

## Slice 1 细化结论回灌（2026-03-11）

### 共识结论（是否通过 + 条件）
- 结论：本轮细化回灌**通过**。
- 单结论：
  - **动态字段统一收口为 4 个非空对象壳加 1 个可空观察位。**
  - **`runtime/transitions.py` 统一只承载状态门禁与版本失效，不承载对象修补、结果映射或编排逻辑。**
  - **`plan_version / finished / blocked / degraded` 的跨对象联动已经形成唯一真值表。**
  - **Guardrails 决策到 `react_status / next_action / final_answer` 的落点已形成唯一映射。**
- 通过条件：
  - 5 个动态字段全部禁止裸 `Any`。
  - 状态矩阵、版本失效、终止态阻断和 `clarify` 禁入主状态规则全部禁止塞进 Pydantic validator。
  - `blocked / degraded / finished` 都不得继续保留动作型 `next_action`。

### 动态字段最终矩阵

| 字段 | 最小类型 | 是否可空 | 默认值 | 约束 |
|---|---|---:|---|---|
| `perception` | `dict[str, object]` | 否 | `{}` | 顶层必须为对象，使用 `default_factory=dict` |
| `stm` | `dict[str, object]` | 否 | `{}` | 顶层必须为对象，使用 `default_factory=dict` |
| `skill_context` | `dict[str, object]` | 否 | `{}` | 无上下文时也保留空对象，使用 `default_factory=dict` |
| `last_observation` | `dict[str, object] | None` | 是 | `None` | 当前唯一允许缺席语义的动态字段 |
| `action_input` | `dict[str, object]` | 否 | `{}` | 缺省必须为对象，使用 `default_factory=dict` |

### `runtime/transitions.py` 四个纯函数契约

| 函数 | 最小输入 | 最小输出 | 错误语义 | 职责边界 |
|---|---|---|---|---|
| `validate_step_transition` | `current: StepState`，`target: StepState` | 返回通过校验后的 `target` | 非法转换抛 `TransitionViolationError` | 只判断 `step_state` 转换是否合法 |
| `validate_cursor_transition` | `current: CursorState`，`target: CursorState` | 返回通过校验后的 `target` | 非法转换抛 `TransitionViolationError` | 只判断 `cursor_state` 转换是否合法 |
| `validate_react_transition` | `current: ReactStatus`，`target: ReactStatus` | 返回通过校验后的 `target` | 非法转换抛 `TransitionViolationError` | 只判断运行态主状态转换，拒绝把 `clarify` 当主状态 |
| `invalidate_current_step_on_plan_change` | `previous_plan_version: str`，`current_plan_version: str`，`current_step: CurrentStep | None` | 未换版返回原 `current_step`，换版返回 `None` | 版本变化本身不报错 | 只负责让旧 `current_step` 失效，不负责推导新 step |

- 固定规则：
  - 4 个函数全部保持无副作用纯函数。
  - 统一采用 keyword-only 签名。
  - 不返回 `bool`，不返回模糊“兼容通过”结果。
  - 不在函数内部做默认值填充、结果映射、编排调度或 `STM` 回写。

### 跨对象一致性真值表

#### `plan_version` 变化
- `current_step`
  - 必须立即失效并置为 `None`。
- `cursor_state`
  - 与旧 `current_step` 一起失效，不自动转成别的合法状态。
- `step_state`
  - 只认新 `Plan.step_state`，不从旧运行态迁移。
- `next_action`
  - 旧计划推导出的动作必须作废，置空或视为无效。
- 报错语义
  - 版本变化本身不报错。
  - 只有继续误用旧 `current_step / cursor_state / next_action` 才报领域异常。

#### `react_status = finished`
- `current_step`
  - 必须置空。
- `next_action`
  - 必须为空。
- `final_answer`
  - 必须存在有效终值。
- `STM`
  - 只允许在 `post_answer` 后回写。
- 其他约束
  - `finished` 为终止态，不允许继续推进新动作。
  - `step_state` 不自动改写，仍由 Planning 权威字段决定。

#### 进入 `blocked / degraded`
- `react_status`
  - 分别落为 `blocked` 或 `degraded`。
- `next_action`
  - 必须为 `None`。
- `final_answer`
  - 必须为 `None`。
- `current_step`
  - 保留当前值但冻结，不再推进。
- `cursor_state`
  - 不自动推进到 `completed`。
- `step_state`
  - 不自动改写。
- 其他约束
  - 本轮不得再推进任何新动作。
  - `tool / mcp / rag / llm / skill` 外部动作链一律停止。

### Guardrails 映射真值表

| 语义 | `react_status` | `next_action` | `final_answer` | 说明 |
|---|---|---|---|---|
| `allow + 动作候选` | `running` | 保留原动作 | `None` | `allow` 不单独编码为状态或动作，只表示放行原候选结果 |
| `allow + 回答候选` | `finished` | `None` | 必须有值 | 允许原回答结果直接收束为完成态 |
| `block` | `blocked` | `None` | `None` | 只允许用 `react_status=blocked` 表达 |
| `degrade` | `degraded` | `None` | `None` | 只允许用 `react_status=degraded` 表达 |
| `clarify` | `running` | `action_type=clarify` | `None` | 只允许落在 `next_action.action_type` |
| `finish` | `finished` | `None` | 必须有值 | 只允许用 `react_status=finished + final_answer` 表达 |

- 明确禁止：
  - `react_status=allow`
  - `react_status=clarify`
  - `blocked / degraded / finished` 仍保留动作型 `next_action`
  - 通过 `final_answer` 并列表达当前阶段的 `block / degrade`
  - 把 `finish` 混入 Guardrails 决策值
