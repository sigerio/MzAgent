# NewAPI 多模型连接处理方案讨论

- 记录时间：2026-03-18
- 用户原始诉求：我举个例子，我有一个newapi的站点，它支持claude模型和gpt模型，我在MzAgent也配置了claude和gpt模型，都是走这个newapi站点进行转发的，那我内部如何处理连接问题呢
- 当前状态：进入方案讨论，未执行代码修改。

## 背景现状

- 用户已明确后续统一通过 NewAPI 提供的 `base_url + api_key + model name` 接入模型。
- 当前讨论已经收敛到：
  - 不再支持官方直连
  - 不再让用户选择供应商
  - 当前内部优先统一走 OpenAI 风格协议
- 用户当前进一步提出典型场景：
  - 一个 NewAPI 网关
  - 同时承载 Claude 模型与 GPT 模型
  - MzAgent 侧需要同时配置多个模型

## 本轮讨论目标

1. 明确“连接配置”和“模型配置”在内部是否应该拆分。
2. 明确同一 NewAPI 站点下多模型共存时的内部数据结构。
3. 明确当前项目短期与中期最合理的处理方案。
