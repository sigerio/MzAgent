# NewAPI 网关与单模型启用方案讨论

- 记录时间：2026-03-18
- 用户原始诉求：连接配置下只需要用户输入base url和api key，然后从站点获取模型，让用户根据需要手动选择需要添加的模型，添加完模型之后，在当前阶段仅支持开启一个模型
- 当前状态：进入方案讨论，未执行代码修改。

## 背景现状

- 用户已明确当前阶段统一通过 NewAPI 网关接入模型。
- 当前已讨论出的方向包括：
  - 不再支持官方直连
  - 不再让用户选择供应商
  - 连接配置与模型配置应逐步拆分
  - 可从 NewAPI 的 `GET /v1/models` 拉取当前可用模型列表
- 用户当前进一步将需求收敛为：
  - 连接配置只录入 `base_url` 与 `api_key`
  - 模型列表从站点拉取
  - 用户手动选择需要添加的模型
  - 当前阶段仅允许启用一个模型

## 本轮讨论目标

1. 明确当前阶段的数据结构应如何拆分“网关”和“模型”。
2. 明确配置页交互是否改为“先连接，后拉模型，再添加模型”。
3. 明确“仅支持开启一个模型”的语义与约束位置。
