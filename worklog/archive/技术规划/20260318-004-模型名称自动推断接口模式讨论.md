# 模型名称自动推断接口模式讨论

- 记录时间：2026-03-18
- 用户原始诉求：能否根据模型名称自动补入messages或者responses等字段
- 当前状态：进入方案讨论，未执行代码修改。

## 背景现状

- 当前项目计划统一通过 NewAPI 提供的 `base_url + api_key + model name` 接入模型。
- 当前代码中存在 `api_mode` 字段，但实际 LLM 调用仍固定走 OpenAI SDK 的 `responses.create(...)`。
- 用户希望进一步简化配置体验，评估是否可以仅根据模型名称自动决定接口模式，例如：
  - `responses`
  - `chat/completions`
  - `messages`

## 本轮讨论目标

1. 讨论是否存在可靠的“模型名 -> 接口模式”自动映射规则。
2. 讨论当前项目是否适合保留 `api_mode` 配置项。
3. 讨论在统一使用 NewAPI 的前提下，推荐的接口模式策略。
