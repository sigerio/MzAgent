# 移除 default 模型选项与空态隐藏

- 记录时间：2026-03-18
- 用户原始诉求：模型列表或者启用的模型中不需要内置默认模型，也就是说不要提供 `default` 选项，如果可用模型为空或者启用模型为空，直接不显示该选项
- 当前状态：已按首版方案完成前端显示改造，待用户确认效果。

## 背景现状

- 当前 Web 设置页存在“可用模型列表”和“启用模型/当前模型”相关选项。
- 用户明确要求移除内置 `default` 选项，不再把它作为兜底项显示给用户。
- 用户同时要求：
  - 若“可用模型列表”为空，则对应下拉或候选区不显示该选项
  - 若“启用模型”为空，则对应下拉或候选区不显示该选项

## 本轮讨论焦点

1. 当前 `default` 选项是在前端静态注入、后端响应兜底，还是两边都有。
2. “不显示该选项”是隐藏整块 UI，还是仅隐藏该条目，需要保持现有布局不抖动。
3. 若移除 `default` 后当前已有旧配置仍保存为 `default`，是否需要在读取时兼容清洗。

## 初步判断

- 该问题大概率集中在前端渲染逻辑与少量状态初始化逻辑：
  - 前端可能存在静态拼接 `default` 条目的实现
  - 状态初始化可能在空数组时写入 `default`
- 后端如未主动注入 `default`，则优先只改前端显示与默认状态逻辑。

## 本轮已执行修改

- 已确认当前所谓 `default` 并非后端返回模型，而是前端在空列表时手工注入的占位 `<option>`。
- 已移除两类空态占位 option：
  - 已添加模型为空时，不再在“当前模型”下拉中注入“当前无配置方案”
  - 可添加模型为空时，不再在“可添加模型”下拉中注入“请先获取模型列表”
- 已新增显隐控制：
  - 顶栏模型切换器在无已添加模型时隐藏
  - 设置抽屉中的“可添加模型”区块在可用模型为空时隐藏
  - 设置抽屉中的“当前模型”区块在已添加模型为空时隐藏
- 已保留原有 `settingsEmptyState` 作为空态提示承载区，不再把提示文字塞进下拉框。
