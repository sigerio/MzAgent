# 添加模型后按钮布局 UI 评审

- 记录时间：2026-03-18
- 用户原始诉求：在添加模型之后，几个按键的布局，位置，通过 ui-ux 测试下
- 当前状态：已完成一轮局部间距微调与预览，待用户确认是否继续调整。

## 背景现状

- 当前设置抽屉在“已添加模型”区域提供三个操作：
  - `启用模型`
  - `测试模型`
  - `删除模型`
- 三个按钮当前处于同一行动作组内，采用横向并列、允许换行的布局。
- 用户要求通过 `ui-ux-pro-max` skill 对该区域进行一次布局与位置评审。

## 本轮讨论焦点

1. 主操作、次操作、危险操作是否已形成足够清晰的视觉优先级。
2. 三个按钮在窄抽屉中的横向排列是否会造成拥挤、误触或阅读顺序混乱。
3. “说明文字”和“结果反馈区域”与按钮组的距离、层次是否合理。

## 本轮已执行调整

- 已对设置抽屉中“下拉框 -> 按钮组”之间增加局部垂直间距。
- 调整范围仅限：
  - “可添加模型”下拉与“添加模型”按钮之间
  - “当前模型”下拉与“启用模型 / 测试模型 / 删除模型”按钮组之间
- 实现方式：
  - 为对应按钮容器增加 `settings-actions-after-select`
  - 在样式中增加 `margin-top: 8px`
