**布局方案**：ChatGPT 风格对话流（用户选定）
- 侧边栏（260px）：品牌标识 + 会话列表 + 能力管理导航 + 设置入口
- 主区域：顶栏 + 对话流（居中 768px）+ 底部输入区
- 设置抽屉：从右滑入，覆盖层模式
- 能力管理抽屉 × 3：TOOLS / MCP / SKILLS 独立管理面板

**文件变更**：
- `index.html` — 全量重写，ChatGPT 风格 HTML 结构
- `styles.css` — 全量重写，暗色主题设计系统（`#0F0F0F` / `#171717` / `#212121`，主色 `#10A37F`）
- `app.js` — 全量重写，模块化结构：DOM 引用 → 状态 → API 封装 → 组件渲染 → 事件绑定

**交互改进**：
- action_type 下拉改为能力开关按钮（工具/MCP/技能/RAG），pill 样式切换
- 开关状态推断 action_type：有开关开启 → `auto`，全关 → `llm`
- 提交时发送 `enabled_capabilities` 数组给后端
- 方案名和显示名合并为单一字段 `profile_name`

**能力管理系统（v0.1.19 新增）**：
- 侧边栏新增「能力管理」导航区，包含 TOOLS / MCP / SKILLS 三个入口按钮（带数量徽章）
- 每个类型独立的右侧管理抽屉（480px），包含：
  - 全局启用/禁用主开关
  - 已启用 / 总计 计数
  - 可折叠的添加表单（各类型有专属字段）
    - TOOLS：名称、描述、调用地址
    - MCP：名称、描述、传输方式（stdio/sse/streamable-http）、启动命令
    - SKILLS：名称、描述、入口路径
  - 列表项：图标 + 名称 + 描述 + 子开关 + hover 显示删除按钮
- 输入区保留 RAG 独立开关（slider 样式）
- 提交时发送细粒度能力列表：`enabled_tools` / `enabled_mcp` / `enabled_skills` / `rag_enabled`
- 使用英文大写命名：TOOLS / MCP / SKILLS（不翻译）

**预留接口**：
- SSE 实时流（`GET /api/agent/stream`，已实现快照式推送，后续改为真正流式）
- WebSocket 双向通信（已注释）
- 多会话列表（HTML 结构已预留）
- Toast 通知系统（已实现）
- 能力 CRUD API（已全部实现）：
  - `GET /api/capabilities/{type}` — 获取能力列表
  - `POST /api/capabilities/{type}` — 添加能力项
  - `DELETE /api/capabilities/{type}/{name}` — 删除能力项
  - `POST /api/capabilities/{type}/{name}/toggle` — 切换启用状态
- 指定轮次重试（已实现）：
  - `POST /api/session/{id}/rounds/{round_id}/retry` — 回退并重跑指定轮次

**启动方式**：`scripts/start_web.sh --host 0.0.0.0 --port 8199`

### 技术说明

- **Markdown 渲染**：纯前端，后端已返回 `content` 字符串，前端引入 marked.js 解析即可
- **消息操作按钮**：「复制」为纯前端 `navigator.clipboard.writeText()`；「重试」后端已实现 `POST /api/session/{id}/rounds/{round_id}/retry`，前端可直接调用；「编辑」仍预留
- **前端对接**：`refreshCapabilities()` 需补全后端返回的 `endpoint/transport/command/entry` 字段；SSE 端点 `/api/agent/stream` 当前为快照式推送，前端先接入基础事件流
- **后续优化**（不在本轮）：SSE 改为真正逐 token 流式推送、retry replay 优化为仅重跑目标轮
