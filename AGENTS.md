# 项目提示词（多项目主仓库）

## 1. 语言与沟通
- 所有回复必须使用中文。
- 用户提出需求后，先确认“当前项目”并讨论方案，不得直接改代码。
- 未获得用户明确同意前，不执行代码修改。
- 用户未指定项目时，必须先询问项目代号或路径，再继续。

## 2. 主仓库定位
- 当前仓库是“多项目管理主仓库”，不是单一业务仓库。
- 该目录下可长期管理多个项目，项目数量与子仓库数量不设上限。
- 每个新项目必须在独立目录中管理，不得与其他项目混放文件。

## 3. 目录与项目登记规范
- 项目统一放在仓库根目录：`<项目代号>/`
- 每个项目至少包含：
  - `<项目代号>/worklog/active/`
  - `<项目代号>/worklog/archive/`
  - `<项目代号>/worklog/index.md`
  - `<项目代号>/worklog/history.md`
  - `<项目代号>/repos.md`（该项目的子仓库与映射登记）
- 子仓库直接放在 `<项目代号>/` 下，不使用固定子目录名。
- 子仓库目录名默认使用克隆后的原始目录名，不做统一改名。
- 子仓库在主仓库中统一以 `git submodule` 管理；主仓库仅追踪子仓库指针与 `.gitmodules`，不追踪子仓库源码文件。
- 主仓库维护项目总登记：`registry.md`（恒短文件，只保留当前态）。
- `registry.md` 仅记录在管项目清单，不记录历史过程，推荐单行格式：
  - `项目目录名 | 一句话描述`
- `registry.md` 不再单独记录项目路径；项目目录名即仓库根目录下的项目文件夹名。
- 项目历史变更、过程讨论、阶段记录一律写入各项目 `worklog/`。
- `<项目代号>/repos.md` 仅记录该项目“当前有效”的子仓库映射与状态，至少包含：
  - 子仓库目录名（克隆原名）
  - 子仓库路径
  - 源码仓库地址（`source_repo`，通常为 `upstream`）
  - 源码分支（`source_branch`，如 `main`）
  - 源码版本（`source_version`，优先 `tag/release`，无则 `commit SHA`）
  - 源码基线提交（`source_base_commit`，当前代码相对源码仓库的基线）
  - 远程地址（`origin/upstream`）
  - 默认分支
  - 最后核验时间（`source_checked_at`）
- 若本轮完成一次上游同步并有效落地（`fetch` + `rebase/merge/cherry-pick` 任一产生实际变更），必须在同轮更新该项目 `repos.md`，且至少更新：`source_version`、`source_base_commit`、`source_checked_at`。
- 禁止在 `registry.md` 和项目 `repos.md` 中累计操作流水或长摘要。

## 4. 项目隔离与防串改
- 同一轮任务只允许操作一个“当前项目”。
- 只允许读取和修改当前项目目录内文件，不得跨项目编辑。
- 禁止在一个任务中混合处理多个项目的代码、日志、提交或推送。
- 若检测到其他项目目录出现意外变更，必须立即停止并询问用户。

## 5. 新项目接入流程
1. 创建项目目录：`<项目代号>/`
2. 初始化项目日志目录与 `worklog/index.md`，并创建项目登记文件 `repos.md`
3. 在项目目录下直接克隆/放置子仓库（保留仓库原目录名）
4. 同步更新 `<项目代号>/repos.md` 与 `registry.md`（含 `source_version/source_base_commit/source_checked_at`）
5. 与用户确认该项目后续默认操作路径

## 6. 日志与归档规则（按项目独立）
- 每个项目仅使用自身 `worklog/`，禁止写入其他项目日志。
- 活跃问题：`worklog/active/<主题>/<YYYYMMDD-序号-标题>.md`
- 已完成问题：`worklog/archive/<主题>/<YYYYMMDD-序号-标题>.md`
- 新问题必须新建文档，不复用旧文档。
- 完成后必须归档，并同步更新该项目 `worklog/index.md` 与 `worklog/history.md`。
- `worklog/index.md` 仅记录当前未完成问题（active）。
- `worklog/history.md` 仅记录已归档问题（archive）。
- 两个索引都只保留“标题 + 路径 + 一句话摘要”，不得写入长过程、命令流水或大段输出。
